  <name>Task 2: IncrementalIndexingService orchestrator</name>

  <action>
Create `com.esmp.indexing.application.IncrementalIndexingService` as a `@Service` class. This is the core orchestrator. It must NOT be `@Transactional` itself — it calls separate transactional methods to ensure delete commits before persist.

**Constructor dependencies:**
- `JavaSourceParser`, `AccumulatorToModelMapper`
- `ClassNodeRepository`, `AnnotationNodeRepository`, `PackageNodeRepository`, `ModuleNodeRepository`, `DBTableNodeRepository`, `BusinessTermNodeRepository`
- `Neo4jClient` (for DETACH DELETE and batch hash lookup)
- `LinkingService`, `RiskService`
- `VectorIndexingService`, `ChunkingService`, `EmbeddingModel`, `QdrantClient`, `VectorConfig`
- `ExtractionConfig`

**Main method: `runIncremental(IncrementalIndexRequest request)` returning `IncrementalIndexResponse`**

Pipeline steps (sequential, NOT wrapped in a single @Transactional):

**Step 1 — Validate:** Resolve sourceRoot (fall back to ExtractionConfig.sourceRoot if blank). Validate sourceRoot directory exists.

**Step 2 — Delete pass** (call a separate `@Transactional("neo4jTransactionManager")` private/package method `deleteClasses(List<String> fqns)`):
- For each FQN in deletedFiles list: first resolve the absolute path to a relative sourceFilePath, then look up the ClassNode FQN by sourceFilePath in Neo4j (Cypher: `MATCH (c:JavaClass) WHERE c.sourceFilePath = $path RETURN c.fullyQualifiedName AS fqn`).
- Execute DETACH DELETE Cypher per FQN:
  ```cypher
  MATCH (c:JavaClass {fullyQualifiedName: $fqn})
  OPTIONAL MATCH (c)-[:DECLARES_METHOD]->(m:JavaMethod)
  OPTIONAL MATCH (c)-[:DECLARES_FIELD]->(f:JavaField)
  DETACH DELETE c, m, f
  ```
- After Neo4j commit, call `vectorIndexingService.deleteByClass(fqn)` for each deleted FQN (Qdrant delete is non-transactional; if it fails, log and continue — orphaned points are harmless and will be overwritten).
- Track count of deleted classes.

**Step 3 — Hash filter:**
- For each changedFile path, compute SHA-256 via `FileHashUtil.sha256()`. Normalize to relative path via `sourceRoot.relativize(absolutePath)`.
- Batch fetch stored contentHash values from Neo4j:
  ```cypher
  MATCH (c:JavaClass) WHERE c.sourceFilePath IN $paths
  RETURN c.sourceFilePath AS path, c.contentHash AS hash
  ```
- Compare: if computed hash equals stored hash, skip the file. Build list of truly-changed paths.
- Track count of skipped files.

**Step 4 — Parse and persist** (call a separate `@Transactional("neo4jTransactionManager")` method `extractAndPersist(List<Path> changedPaths, Path sourceRootPath, String classpathFile, Map<String, String> fileHashMap)` returning extraction counts):
- Parse only the truly-changed files: `javaSourceParser.parse(changedPaths, sourceRootPath, classpathFile)`
- Run all 7 visitors on parsed source files (same order as ExtractionService)
- Map accumulator to model entities
- Inject contentHash into each ClassNode BEFORE saveAll: iterate mapped ClassNode list, call `classNode.setContentHash(fileHashMap.get(classNode.getSourceFilePath()))`
- saveAll for all entity types (classNodes, annotationNodes, packageNodes, moduleNodes, dbTableNodes)
- Persist business term nodes using curated-guard MERGE Cypher (duplicate the pattern from ExtractionService.persistBusinessTermNodes — it's private)
- Return the ExtractionAccumulator and counts for response building

**Step 5 — Global link:** `linkingService.linkAllRelationships(accumulator)` — always run, even if no files changed (deletion may have removed edges that need re-evaluation). Note: `linkAllRelationships` takes an `ExtractionAccumulator` but the MERGE queries are global (they MERGE across all nodes in Neo4j, not just those in the accumulator). Pass the accumulator from Step 4, or a fresh empty one if no files were extracted.

**Step 6 — Global risk:** `riskService.computeAndPersistRiskScores()` — always run after linking.

**Step 7 — Selective vector re-embed:**
- Collect FQNs of all truly-changed classes (from Step 4 extraction results)
- For each changed FQN: `vectorIndexingService.deleteByClass(fqn)` (remove old chunks)
- Call `chunkingService.chunkByFqns(changedFqns, sourceRoot)` to get chunks for only changed classes
- Embed and upsert: Use `embeddingModel.embed(texts)` + build PointStruct + `qdrantClient.upsertAsync()` in batches (replicate the embedAndUpsert pattern from VectorIndexingService, or make VectorIndexingService.embedAndUpsert package-accessible by removing `private`).
- Track chunksReEmbedded and chunksDeleted counts.

**Step 8 — Build response:** Return IncrementalIndexResponse with all counts and durationMs.

**Error handling:** Wrap each step in try-catch. Accumulate errors in a List<String>. If delete fails for one FQN, log and continue with remaining. If parse fails for one file, log and continue. Return partial results with error list populated.

**CRITICAL transactional note:** The `runIncremental` method itself must NOT have `@Transactional`. The delete and extract-persist steps must be in separate `@Transactional("neo4jTransactionManager")` methods so delete commits before persist starts. This avoids SDN session cache version conflicts (see Research Pitfall 1). Use Spring's default propagation: since the caller is NOT transactional, each callee's @Transactional creates and commits its own transaction.
