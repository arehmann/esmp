    **Response DTOs (Java records in com.esmp.graph.api):**

    ClassStructureResponse: record with fields:
    - String fullyQualifiedName, String simpleName, String packageName
    - List<String> labels (dynamic labels like Service, Repository, VaadinView)
    - List<String> annotations
    - String superClass
    - List<String> implementedInterfaces
    - List<MethodSummary> methods — record: String methodId, String simpleName, String returnType, List<String> parameterTypes, List<String> annotations
    - List<FieldSummary> fields — record: String fieldId, String simpleName, String fieldType, List<String> annotations
    - List<DependencySummary> dependencies — record: String fullyQualifiedName, String simpleName, String injectionType
    - List<String> annotationNodes — FQNs of AnnotationNode entities linked via HAS_ANNOTATION

    InheritanceChainResponse: record with fields:
    - String fullyQualifiedName (the queried class)
    - List<AncestorEntry> chain — record: String fullyQualifiedName, String simpleName, int depth (1 = direct parent, 2 = grandparent, etc.)
    - List<String> implementedInterfaces (all interfaces in the chain)

    DependencyResponse: record with fields:
    - String repositoryFqn (the queried repository)
    - List<ServiceEntry> services — record: String fullyQualifiedName, String simpleName, int hops (1 = direct, 2+ = transitive)

    SearchResponse: record with fields:
    - String query
    - List<SearchEntry> results — record: String fullyQualifiedName, String simpleName, String packageName, List<String> labels

    **GraphQueryRepository (com.esmp.graph.persistence):**
    Interface extending Neo4jRepository<ClassNode, String> (reuses ClassNode entity — this is a read-only query interface, not for writes).
    - Optional<ClassNode> findByFullyQualifiedName(String fqn) — SDN derived query
    - List<ClassNode> findBySimpleNameContainingIgnoreCase(String name) — for search

    **GraphQueryService (com.esmp.graph.application):**
    @Service class with constructor-injected Neo4jClient and GraphQueryRepository.

    1. findClassStructure(String fqn) -> Optional<ClassStructureResponse>:
       Use Neo4jClient with Cypher:
       ```
       MATCH (c:JavaClass {fullyQualifiedName: $fqn})
       OPTIONAL MATCH (c)-[:DECLARES_METHOD]->(m:JavaMethod)
       OPTIONAL MATCH (c)-[:DECLARES_FIELD]->(f:JavaField)
       OPTIONAL MATCH (c)-[:DEPENDS_ON]->(dep:JavaClass)
       OPTIONAL MATCH (c)-[:HAS_ANNOTATION]->(ann:JavaAnnotation)
       RETURN c, collect(DISTINCT m) AS methods, collect(DISTINCT f) AS fields,
              collect(DISTINCT dep) AS dependencies, collect(DISTINCT ann) AS annotations
       ```
       Map result manually using mappedBy() lambda. Return Optional.empty() if no class found.

    2. findInheritanceChain(String fqn) -> InheritanceChainResponse:
       Use Neo4jClient with Cypher (variable-length path — MUST use Neo4jClient, not @Query):
       ```
       MATCH (c:JavaClass {fullyQualifiedName: $fqn})
       OPTIONAL MATCH path = (c)-[:EXTENDS*1..10]->(ancestor:JavaClass)
       WITH c, nodes(path) AS chainNodes
       UNWIND chainNodes AS node
       WITH c, node WHERE node <> c
       RETURN c.fullyQualifiedName AS rootFqn, c.simpleName AS rootName,
              collect(DISTINCT {fqn: node.fullyQualifiedName, name: node.simpleName}) AS ancestors
       ```
       IMPORTANT: Bound depth to *1..10 for performance safety. Map ancestors into AncestorEntry list with depth computed from position. If no ancestors found, return empty chain.

    3. findServiceDependents(String repositoryFqn) -> DependencyResponse:
       Use Neo4jClient with Cypher:
       ```
       MATCH (repo:JavaClass {fullyQualifiedName: $fqn})
       WHERE repo:Repository OR ANY(label IN labels(repo) WHERE label = 'Repository')
       MATCH path = (svc:JavaClass)-[:DEPENDS_ON*1..10]->(repo)
       WHERE ANY(label IN labels(svc) WHERE label = 'Service')
       RETURN DISTINCT svc.fullyQualifiedName AS fqn, svc.simpleName AS name,
              length(path) AS hops
       ORDER BY hops ASC
       ```
       Map results into ServiceEntry list. If repo not found or no dependents, return empty list.

    4. searchByName(String name) -> SearchResponse:
       Use graphQueryRepository.findBySimpleNameContainingIgnoreCase(name). Map results to SearchEntry list including dynamic labels.

    All Neo4jClient queries use .bind(param).to("paramName") for parameterization — never string concatenation.

    **Create GraphQueryController** (com.esmp.graph.api):
    @RestController @RequestMapping("/api/graph")

    Endpoints:
    1. @GetMapping("/class/{fqn}") getClassStructure(@PathVariable String fqn) -> ResponseEntity<ClassStructureResponse>
       - Delegates to graphQueryService.findClassStructure(fqn)
       - Returns 200 with body, or 404 if Optional.empty()
       - NOTE: FQN contains dots (com.example.Foo). Spring MVC truncates path variables at dots by default. Use @GetMapping("/class/{fqn:.+}") or @PathVariable with regex to preserve full FQN. Alternatively, accept FQN as a @RequestParam instead.

    2. @GetMapping("/class/{fqn:.+}/inheritance") getInheritanceChain(@PathVariable String fqn) -> ResponseEntity<InheritanceChainResponse>
       - Delegates to graphQueryService.findInheritanceChain(fqn)
       - Always returns 200 (empty chain if class not found or no ancestors)

    3. @GetMapping("/repository/{fqn:.+}/service-dependents") getServiceDependents(@PathVariable String fqn) -> ResponseEntity<DependencyResponse>
       - Delegates to graphQueryService.findServiceDependents(fqn)
       - Always returns 200

    4. @GetMapping("/search") search(@RequestParam String name) -> ResponseEntity<SearchResponse>
       - Delegates to graphQueryService.searchByName(name)
       - Always returns 200
