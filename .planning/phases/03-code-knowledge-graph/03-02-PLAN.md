    **Create LinkingService** in com.esmp.extraction.application:

    Uses Neo4jClient for raw Cypher MERGE queries. All methods @Transactional("neo4jTransactionManager").

    linkAllRelationships(ExtractionAccumulator acc) -> LinkingResult:
    Calls each linking method in sequence and aggregates counts.

    1. linkInheritanceRelationships(): EXTENDS edges via Cypher MERGE matching child.superClass to parent.fullyQualifiedName. IMPLEMENTS edges via UNWIND child.implementedInterfaces. Both set resolutionConfidence='RESOLVED'. Unresolved external types are silently skipped (MATCH finds nothing).

    2. linkDependencies(ExtractionAccumulator acc): For each DependencyEdge in acc, use Cypher MERGE to create DEPENDS_ON between source and target ClassNode. Set injectionType and fieldName properties. First delete existing DEPENDS_ON edges for idempotency: `MATCH (:JavaClass)-[r:DEPENDS_ON]->(:JavaClass) DELETE r` then create fresh.

    3. linkTableMappings(ExtractionAccumulator acc): For each table mapping in acc, MERGE MAPS_TO_TABLE edge from ClassNode to DBTableNode.

    4. linkQueryMethods(ExtractionAccumulator acc): For each QueryMethodRecord, find the method's declaring class, look up its table mappings, and MERGE QUERIES edge from MethodNode to DBTableNode. Note: the declaring class must be a Repository (has @Repository or extends a Spring Data interface). The table is found by looking at the generic type parameter of the repository interface — this requires the repository interface's type hierarchy. For simplicity: if the declaring class has table mappings via acc.getTableMappings(), use those. Otherwise, skip with WARN log.

    5. linkPackageHierarchy(ExtractionAccumulator acc): MERGE CONTAINS_CLASS from PackageNode to ClassNode by matching packageName. MERGE CONTAINS_PACKAGE from ModuleNode to PackageNode by matching moduleName.

    6. linkAnnotations(ExtractionAccumulator acc): For each class in acc.getClasses(), iterate its annotations list and MERGE HAS_ANNOTATION edge from ClassNode to AnnotationNode by matching annotation FQN.

    Return LinkingResult record with counts for each edge type.

    **CRITICAL ordering (per Research Pitfall 2):** LinkingService methods MUST run after classNodeRepository.saveAll() and all other repository saveAll() calls have completed. The ExtractionService.extract() method must be restructured so that node persistence happens first, then linkingService runs in a separate step. Two approaches:
    - Option A: Remove @Transactional from extract(), use programmatic transaction boundaries
    - Option B: Have linking run as a separate @Transactional method called after extract(). The caller (controller) orchestrates both.
    Choose the approach that is simplest. Since ExtractionService.extract() is already @Transactional, and the linking Cypher runs in the same Neo4j database, the MERGE Cypher inside the same transaction should see the nodes saved by saveAll() within that same transaction (SDN flushes to the database session). Test this — if linking finds 0 edges, restructure into two transactions.
