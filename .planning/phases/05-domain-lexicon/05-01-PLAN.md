    1. Add `mapToBusinessTermNodes(ExtractionAccumulator acc)` method to `AccumulatorToModelMapper`:
       - Iterate `acc.getBusinessTerms().values()`
       - For each BusinessTermData, create a BusinessTermNode:
         - termId = data.termId
         - displayName = data.displayName (capitalize first letter)
         - definition = data.javadocSeed (null is fine — blank definition per decision)
         - criticality = heuristic seed: check if termId matches financial keywords (payment, invoice, billing, ledger, transaction, price, cost, fee, amount, currency, refund, credit, debit) -> "High"; auth/security keywords (auth, login, password, credential, token, session, permission, role, encrypt, decrypt, security, access) -> "High"; otherwise "Low"
         - migrationSensitivity = if criticality is "High" then "Moderate" else "None"
         - curated = false
         - status = "auto"
         - sourceType = data.sourceType
         - primarySourceFqn = data.primarySourceFqn
         - usageCount = data.allSourceFqns.size()
       - Return List of BusinessTermNode

    2. Add DB table/column term extraction as post-visitor step in `mapToBusinessTermNodes` or as a separate helper. Since LexiconVisitor runs during the visitor pass but DB table names come from `acc.getTableMappings()` (populated by JpaPatternVisitor), process table names in the mapper:
       - For each entry in `acc.getTableMappings()`: split table name on underscores, filter len>2, add as BusinessTermNode with sourceType "DB_TABLE", primarySourceFqn = the entity class FQN
       - Merge with visitor-extracted terms by termId (skip if already exists from visitor pass)

    3. Wire into `ExtractionService.extract()`:
       - Add `BusinessTermNodeRepository` as constructor dependency
       - Create `LexiconVisitor lexiconVisitor = new LexiconVisitor();` alongside other visitors
       - Add `lexiconVisitor.visit(sourceFile, accumulator);` in the visitor loop (after jpaPatternVisitor to ensure table mappings exist)
       - After mapper calls, add: `List<BusinessTermNode> businessTermNodes = mapper.mapToBusinessTermNodes(accumulator);`
       - Persist: Use Neo4jClient with the curated-guard MERGE Cypher instead of saveAll() to prevent overwriting curated terms:
         ```
         MERGE (t:BusinessTerm {termId: $termId})
         ON CREATE SET t.displayName=$displayName, t.definition=$definition, t.criticality=$criticality,
           t.migrationSensitivity=$sensitivity, t.curated=false, t.status='auto',
           t.sourceType=$sourceType, t.primarySourceFqn=$fqn, t.usageCount=$usageCount, t.synonyms=[]
         ON MATCH SET
           t.displayName = CASE WHEN t.curated THEN t.displayName ELSE $displayName END,
           t.definition = CASE WHEN t.curated THEN t.definition ELSE $definition END,
           t.criticality = CASE WHEN t.curated THEN t.criticality ELSE $criticality END,
           t.usageCount = $usageCount,
           t.status = CASE WHEN t.curated THEN 'curated' ELSE 'auto' END
         ```
       - Log: `log.info("Persisted {} business term nodes to Neo4j", businessTermNodes.size());`
       - Update ExtractionResult record: add `int businessTermCount` field, update constructor call

    IMPORTANT: The curated-guard MERGE is the core LEX-02/LEX-04 compliance mechanism. Do NOT use simple `saveAll()` which would overwrite curated definitions. Use Neo4jClient for this specific persistence step.
