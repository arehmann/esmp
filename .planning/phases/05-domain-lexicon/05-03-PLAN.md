       - Constructor-injects `LexiconService`
       - Loads all terms via `lexiconService.findAll()`
       - Uses `ListDataProvider<BusinessTermNode>` for in-memory filtering/sorting
       - Grid columns (all sortable except definition and actions):

         - "Criticality" — criticality, sortable
         - "Source" — sourceType, sortable
         - "Usage Count" — usageCount, sortable, rendered as a clickable Anchor/Button showing the count (e.g., "12 classes")
         - "Status" — curated ? "Curated" : "Auto", sortable
         - "Actions" — component column with Edit button

         - On click: call `lexiconService.findByTermId(term.getTermId())` to fetch the `BusinessTermResponse` with its `relatedClassFqns` list
         - Open `TermDetailDialog(term.getDisplayName(), response.relatedClassFqns())` showing the list of related FQNs
       - Filter row (appended header row):
         - TextField in "Term" column header: filters by displayName case-insensitive contains

         - ComboBox in "Status" column header: items Curated/Auto/null(All), filters by curated boolean
       - IMPORTANT: Use `dataProvider.setFilter()` or clear-and-re-apply pattern to avoid filter accumulation (pitfall 4 from research). Maintain a single composite predicate field, update on each filter change, call `dataProvider.refreshAll()`.

         - Show success notification via `Notification.show("Term updated")`
       - Grid should be `setSizeFull()`, layout should be `setSizeFull()` and `setPadding(true)`
