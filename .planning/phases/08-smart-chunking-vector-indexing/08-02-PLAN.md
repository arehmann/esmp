2. **VectorIndexingService.java**: `@Service` in `com.esmp.vector.application`. Inject `EmbeddingModel`, `QdrantClient`, `ChunkingService`, `VectorConfig`.

   **indexAll(String sourceRoot) method:**
   - Call `chunkingService.chunkClasses(sourceRoot)` to get all chunks.
   - Process in batches of `vectorConfig.getBatchSize()`:
     a. Extract text from each chunk in the batch.
     b. Call `embeddingModel.embed(texts)` to get embeddings. The Spring AI EmbeddingModel.embed(List of String) returns an EmbeddingResponse — extract the float arrays from it. Convert double arrays to float arrays for Qdrant.
     c. Build `List<PointStruct>` using Qdrant factories:
        - `PointIdFactory.id(chunk.pointId())` for the UUID point ID
        - `VectorsFactory.vectors(floatArray)` for the embedding
        - `ValueFactory.value(...)` for each payload field:
          - classFqn (String), chunkType (String from enum name), methodId (String, empty for header), classHeaderId (String), module (String), stereotype (String), contentHash (String)
          - structuralRiskScore, enhancedRiskScore, domainCriticality, securitySensitivity, financialInvolvement, businessRuleDensity (all double)
          - vaadin7Detected (boolean)
          - callers, callees, dependencies, implementors (comma-joined String)
          - domainTerms (JSON string: serialize List of DomainTermRef as `[{"termId":"...","displayName":"..."},...]`)
     d. Call `qdrantClient.upsertAsync(collectionName, points).get(30, SECONDS)`
   - Return `IndexStatusResponse` with counts and duration.

   **reindex(String sourceRoot) method (incremental):**
   - Step 1: Scroll all existing points in Qdrant to build `Map<String, String>` of classFqn -> contentHash (stored hash). Use `scrollAsync` with `WithPayloadSelectorFactory.include(List.of("classFqn", "contentHash"))`, page size 500, paginating via `getNextPageOffset()` (use the PointId directly as offset, do NOT `.toString()` it — see Pitfall 6 in RESEARCH.md).
   - Step 2: Call `chunkingService.chunkClasses(sourceRoot)` to get current chunks.
   - Step 3: Group chunks by classFqn. For each class, compare current contentHash with stored hash. If same, skip all chunks for that class. If different (or class not in stored map), include all chunks for re-embedding.
   - Step 4: Embed and upsert only changed chunks (same batch logic as indexAll).
   - Step 5: Return `IndexStatusResponse` with counts (filesProcessed = changed classes, chunksSkipped = unchanged chunks).

   **deleteByClass(String classFqn) method** (for future use):
   - Use Qdrant filter to find points where classFqn matches, then delete them. (Keep simple for now.)

   **Error handling:** Wrap individual batch upserts in try/catch. Log failures per batch but continue processing remaining batches. Count failed chunks as `chunksSkipped`.

3. **VectorIndexController.java**: `@RestController @RequestMapping("/api/vector")` in `com.esmp.vector.api`. Inject `VectorIndexingService`.
   - `POST /api/vector/index` — accepts `@RequestParam String sourceRoot`, calls `indexAll(sourceRoot)`, returns `IndexStatusResponse`.
   - `POST /api/vector/reindex` — accepts `@RequestParam String sourceRoot`, calls `reindex(sourceRoot)`, returns `IndexStatusResponse`.
   - Both return 200 with the response record. Wrap in try/catch and return 500 with error message on failure.

1. **VectorValidationQueryRegistry.java**: `@Component` in `com.esmp.vector.validation`, extends `ValidationQueryRegistry`. Provide 3 validation queries in the constructor:
   - `VECTOR_INDEX_POPULATED`: Cypher query that counts JavaClass nodes with non-null sourceFilePath and compares against Qdrant collection point count (this is a hybrid check — query Neo4j for expected count, note: actual Qdrant check happens at REST level, so this query just checks the Neo4j side: `MATCH (c:JavaClass) WHERE c.sourceFilePath IS NOT NULL RETURN count(c) AS expected`). Set pass condition as `count > 0`.
   - `VECTOR_CHUNKS_HAVE_CONTENT_HASH`: Cypher: `MATCH (c:JavaClass) WHERE c.sourceFilePath IS NOT NULL AND c.contentHash IS NULL RETURN count(c) AS missing_hashes` — PASS if count = 0.
   - `VECTOR_SOURCE_FILES_ACCESSIBLE`: Cypher: `MATCH (c:JavaClass) WHERE c.sourceFilePath IS NOT NULL RETURN count(c) AS total, collect(c.sourceFilePath) AS paths` — informational query to verify source paths are populated.
