**3. Implement `detectTransitiveMigrations()`** — returns count of inherited actions created:

Step 3a — Get all known Vaadin 7 source FQNs from the recipe book:
```java
List<String> vaadinSourceFqns = recipeBookRegistry.getRules().stream()
    .filter(r -> !"NEEDS_MAPPING".equals(r.status()))
    .filter(r -> r.source().startsWith("com.vaadin.") && !r.source().startsWith("com.vaadin.flow."))
    .map(RecipeRule::source)
    .distinct()
    .collect(Collectors.toList());
```

Step 3b — Cypher to find transitive inheritors (non-direct, via EXTENDS):
```cypher
MATCH (c:JavaClass)-[:EXTENDS*1..10]->(ancestor:JavaClass)
WHERE ancestor.fullyQualifiedName IN $vaadinSourceFqns
  AND NOT (c)-[:HAS_MIGRATION_ACTION]->(:MigrationAction {source: ancestor.fullyQualifiedName})
RETURN c.fullyQualifiedName AS classFqn,
       ancestor.fullyQualifiedName AS ancestorFqn,
       labels(c) AS classLabels
```

Step 3c — For each result, compute complexity profile via a second Cypher query:
```cypher
OPTIONAL MATCH (ancestor:JavaClass {fullyQualifiedName: $ancestorFqn})-[:DECLARES_METHOD]->(am:JavaMethod)
WITH collect(DISTINCT am.simpleName) AS ancestorMethodNames
OPTIONAL MATCH (c:JavaClass {fullyQualifiedName: $classFqn})-[:DECLARES_METHOD]->(cm:JavaMethod)
WHERE cm.simpleName IN ancestorMethodNames
WITH count(cm) AS overrideCount, ancestorMethodNames
OPTIONAL MATCH (c:JavaClass {fullyQualifiedName: $classFqn})-[:CALLS]->(callee:JavaClass)
WHERE callee.fullyQualifiedName STARTS WITH 'com.vaadin.'
  AND NOT callee.fullyQualifiedName STARTS WITH 'com.vaadin.flow.'
WITH overrideCount, count(DISTINCT callee) AS ownVaadinCalls
RETURN overrideCount, ownVaadinCalls
```
- Also check `classLabels` from step 3b for `VaadinDataBinding` and `VaadinComponent`

Step 3d — Compute transitiveComplexity score using MigrationConfig.TransitiveConfig weights:
```java
double rawScore = (overrideCount * config.getTransitive().getOverrideWeight())
    + (ownVaadinCalls * config.getTransitive().getOwnCallsWeight())
    + (hasOwnBinding ? config.getTransitive().getBindingWeight() : 0.0)
    + (hasOwnComponents ? config.getTransitive().getComponentWeight() : 0.0);
double transitiveComplexity = Math.min(1.0, rawScore);
boolean pureWrapper = transitiveComplexity == 0.0;
String automatable;
if (pureWrapper) {
    automatable = "PARTIAL";
} else if (transitiveComplexity <= config.getTransitive().getAiAssistedThreshold()) {
    automatable = "PARTIAL";
} else {
    automatable = "NO";
}
```

Step 3e — MERGE inherited MigrationAction node + HAS_MIGRATION_ACTION edge via Cypher:
```cypher
MERGE (ma:MigrationAction {actionId: $actionId})
ON CREATE SET ma.classFqn = $classFqn, ma.actionType = 'COMPLEX_REWRITE',
              ma.source = $ancestorFqn, ma.target = $target,
              ma.automatable = $automatable, ma.context = $context,
              ma.isInherited = true, ma.pureWrapper = $pureWrapper,
              ma.transitiveComplexity = $transitiveComplexity,
              ma.vaadinAncestor = $ancestorFqn,
              ma.overrideCount = $overrideCount, ma.ownVaadinCalls = $ownVaadinCalls
ON MATCH SET  ma.automatable = $automatable, ma.transitiveComplexity = $transitiveComplexity,
              ma.pureWrapper = $pureWrapper, ma.overrideCount = $overrideCount,
              ma.ownVaadinCalls = $ownVaadinCalls
WITH ma
MATCH (c:JavaClass {fullyQualifiedName: $classFqn})
MERGE (c)-[:HAS_MIGRATION_ACTION]->(ma)
```
Where `actionId = classFqn + "#INHERITED#" + ancestorFqn`
Where `target` comes from `recipeBookRegistry.findBySource(ancestorFqn).map(RecipeRule::target).orElse("Manual migration required")`
Where `context = "Inherited from " + ancestorFqn + (pureWrapper ? " (pure wrapper — mechanical wrapping)" : " (complex — overrides or own Vaadin usage)")`

**4. Implement `recomputeMigrationScores()`** — updates ClassNode migration properties:
```cypher
MATCH (c:JavaClass)
OPTIONAL MATCH (c)-[:HAS_MIGRATION_ACTION]->(ma:MigrationAction)
WITH c, count(ma) AS total,
     sum(CASE WHEN ma.automatable = 'YES' THEN 1 ELSE 0 END) AS yesCount,
     sum(CASE WHEN ma.automatable = 'PARTIAL' THEN 1 ELSE 0 END) AS partialCount,
     sum(CASE WHEN ma.automatable = 'NO' THEN 1 ELSE 0 END) AS noCount
WHERE total > 0
SET c.migrationActionCount = total,
    c.automatableActionCount = yesCount,
    c.automationScore = CASE WHEN total > 0 THEN (toFloat(yesCount) + 0.5 * partialCount) / total ELSE 0.0 END,
    c.needsAiMigration = (noCount > 0)
```

**5. Implement `enrichRecipeBook()`** — usage count aggregation + NEEDS_MAPPING discovery:

Step 5a — Aggregate usage counts by source FQN:
```cypher
MATCH (ma:MigrationAction)
WHERE NOT ma.isInherited
RETURN ma.source AS source, count(ma) AS usageCount
```

Step 5b — Find unmapped types not in recipe book:
```cypher
MATCH (ma:MigrationAction)
WHERE ma.source STARTS WITH 'com.vaadin.'
  AND NOT ma.source STARTS WITH 'com.vaadin.flow.'
  AND ma.automatable = 'NO'
  AND ma.actionType = 'COMPLEX_REWRITE'
  AND ma.context CONTAINS 'Unknown Vaadin 7 type'
RETURN DISTINCT ma.source AS source, count(ma) AS usageCount
ORDER BY usageCount DESC
```

Step 5c — Update recipe book in memory:
- Clone current rules from `recipeBookRegistry.getRules()` into a mutable list
- For each usage count result, find matching rule by source and update its `usageCount`
- For each unmapped type not already in the list, add a new `RecipeRule` with:
  - id: "DISC-" + 3-digit sequential number (count existing DISCOVERED rules + 1)
  - category: "DISCOVERED"
  - source: the unmapped FQN
  - target: null
  - actionType: "COMPLEX_REWRITE"
  - automatable: "NO"
  - context: null
  - migrationSteps: empty list
  - status: "NEEDS_MAPPING"
  - usageCount: from query
  - discoveredAt: LocalDate.now().toString()
  - isBase: false

Step 5d — Write back via `recipeBookRegistry.updateAndWrite(updatedRules)` wrapped in try/catch. IOException is logged as WARNING (non-fatal). The recipe book is a cache of graph data — stale counts until next extraction is acceptable.

**IMPORTANT:** `isBase` field must NOT be serialized to JSON. The `RecipeBook` JSON format does not include `isBase`. When `RecipeBookRegistry.updateAndWrite()` serializes, it should use a Jackson `@JsonIgnore` on `isBase` in the `RecipeRule` record, OR create a version without `isBase` for serialization. Simplest approach: use `@JsonIgnore` on the `isBase` component getter, or use a `@JsonProperty(access = READ_ONLY)` approach. Since records don't support `@JsonIgnore` on components easily, recommend adding a Jackson `MixIn` or converting `RecipeRule` to have `@JsonInclude(Include.NON_DEFAULT)` on isBase (default false, so false won't serialize). The `RecipeBookRegistry` can also manually strip `isBase` when writing by mapping to a new record without it. Executor should choose the cleanest approach.
