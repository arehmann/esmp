<objective>
Add fan-in/out computation, composite risk scoring, and REST API endpoints for the structural risk heatmap.

Purpose: RISK-02 (fan-in/fan-out metrics), RISK-04 (composite structural risk score), and RISK-05 (dependency heatmap endpoint) complete the structural risk analysis feature. Fan-in/out are computed from existing DEPENDS_ON edges via Cypher aggregation (no re-parsing). The composite score is a configurable weighted sum. The REST API provides filterable heatmap and per-class detail endpoints.

Output: RiskService, RiskController, RiskWeightConfig, RiskValidationQueryRegistry, response records, schema index, integration tests.

       - Inject `Neo4jClient` and `RiskWeightConfig`
       - **`computeAndPersistRiskScores()`** method:
         - Step 1: Compute fan-in/out via single Cypher query:
           ```cypher
           MATCH (c:JavaClass)
           WITH c,
                size([(other)-[:DEPENDS_ON]->(c) | other]) AS fi,
                size([(c)-[:DEPENDS_ON]->(other) | other]) AS fo
           SET c.fanIn = fi, c.fanOut = fo
           ```
         - Step 2: Compute composite score via single Cypher query using log normalization:
           ```cypher
           MATCH (c:JavaClass)
           SET c.structuralRiskScore = (
               $wComplexity * log(1.0 + coalesce(c.complexitySum, 0)) +
               $wFanIn * log(1.0 + coalesce(c.fanIn, 0)) +
               $wFanOut * log(1.0 + coalesce(c.fanOut, 0)) +
               $wDbWrites * CASE WHEN c.hasDbWrites THEN 1.0 ELSE 0.0 END
           )
           ```
           Bind weights from RiskWeightConfig. Use `neo4jClient.query(cypher).bindAll(Map.of(...)).run()`.
       - **`getHeatmap(String module, String packageName, String stereotype, int limit)`**:
         - Build Cypher dynamically based on which filters are non-null:
           ```cypher
           MATCH (c:JavaClass)
           WHERE c.structuralRiskScore IS NOT NULL
           [AND c.packageName STARTS WITH $packageName]
           [AND ANY(label IN labels(c) WHERE label = $stereotype)]
           RETURN c
           ORDER BY c.structuralRiskScore DESC
           LIMIT $limit
           ```
         - For module filter: `MATCH (m:JavaModule {moduleName: $module})-[:CONTAINS_PACKAGE]->(p:JavaPackage)-[:CONTAINS_CLASS]->(c:JavaClass)`
         - Map results using `neo4jClient.query(cypher).bindAll(params).fetchAs(RiskHeatmapEntry.class).mappedBy((ts, record) -> mapNodeToHeatmapEntry(record.get("c").asNode())).all()`
         - `mapNodeToHeatmapEntry(Node node)`: use `node.get("prop").asXxx(default)` pattern from LexiconService. Extract stereotype labels from `node.labels()` filtering for known stereotypes (Service, Repository, VaadinView, etc.).
       - **`getClassDetail(String fqn)`** returning `Optional<RiskDetailResponse>`:
         - Cypher: match class by FQN, collect its methods with cyclomaticComplexity:
           ```cypher
           MATCH (c:JavaClass {fullyQualifiedName: $fqn})
           OPTIONAL MATCH (c)-[:DECLARES_METHOD]->(m:JavaMethod)
           RETURN c, collect(m) AS methods
           ```
         - Map class node to RiskDetailResponse with method list as MethodComplexityEntry records.
