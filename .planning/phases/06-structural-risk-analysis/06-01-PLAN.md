    4. **Create ComplexityVisitor.java** — `JavaIsoVisitor<ExtractionAccumulator>`:
       - **Complexity counting strategy**: Use a `Deque<MutableInt>` stack (or `Deque<int[]>`) to track per-method branch counts. Push a new counter on `visitMethodDeclaration` entry (before `super` call), pop and commit on exit (after `super` call). This safely handles nested methods/lambdas without instance-state leaking between source files.
       - Override `visitMethodDeclaration`: push counter, call super (which recurses into body), pop counter, compute CC = counter + 1 (baseline), call `acc.addMethodComplexity(methodId, declaringClassFqn, cc)`. Use `ClassMetadataVisitor.buildMethodId()` to build the methodId.
       - Override `visitIf`: increment top counter +1; call super. Do NOT count `J.If.Else` separately -- `else if` is a nested `J.If` which will be visited and counted.
       - Override `visitTernary`: increment top counter +1; call super.
       - Override `visitWhileLoop`: increment top counter +1; call super.
       - Override `visitDoWhileLoop`: increment top counter +1; call super.
       - Override `visitForLoop`: increment top counter +1; call super.
       - Override `visitForEachLoop`: increment top counter +1; call super.
       - Override `visitCase`: increment top counter +1 for each case label (but NOT the default case -- `J.Case` with `getExpressions()` empty or `getPattern()` being `J.DefaultType` is a default); call super.
       - Override `visitCatch`: increment top counter +1; call super. Access via `J.Try.Catch`.
       - **DB write detection** (in the same visitor for efficiency):
         - Override `visitMethodDeclaration`: after super call, also check annotations for `@Modifying` (FQN: `org.springframework.data.jpa.repository.Modifying`; simple-name fallback: `"Modifying"`). If found, call `acc.incrementClassDbWrites(declaringClassFqn)`.
         - Override `visitAnnotation`: if annotation is `@Query`, check the annotation argument string literal for `INSERT|UPDATE|DELETE` keywords (case-insensitive). If match, call `acc.incrementClassDbWrites(declaringClassFqn)` using `getCursor().firstEnclosing(J.ClassDeclaration.class)` to find the class FQN.
         - Override `visitMethodInvocation`: check if method name is `persist`, `merge`, `remove`, `delete`, `save`, `saveAll`, `deleteAll`, `flush`. If the receiver type is `EntityManager` or if the method name matches these JPA/Spring Data write methods, call `acc.incrementClassDbWrites(declaringClassFqn)` using cursor to find enclosing class.
       - **CRITICAL**: Use annotation FQN resolution with simple-name fallback (same pattern as JpaPatternVisitor.resolveAnnotationFqn). Add `"Modifying" -> "org.springframework.data.jpa.repository.Modifying"` to the fallback switch.
