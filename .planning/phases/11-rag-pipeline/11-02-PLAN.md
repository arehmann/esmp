Create `com.esmp.rag.application.RagService` as a `@Service` class. This is a pure orchestrator — it does NOT write to Neo4j or JPA, so do NOT annotate with `@Transactional`.

**Constructor dependencies:**
- `Neo4jClient neo4jClient` — for hop-distance cone Cypher query
- `VectorSearchService vectorSearchService` — for `searchByCone()`
- `EmbeddingModel embeddingModel` — Spring AI, for embedding focal class text
- `GraphQueryService graphQueryService` — for `searchByName()` (query resolution)
- `RagWeightConfig ragWeightConfig` — for ranking weights

**Main public method: `assemble(RagRequest request) -> RagResponse`**

Implementation flow:

**Step 1 — Query Resolution (3 paths):**

Path A: If `request.fqn()` is non-null and non-blank, use it directly as the resolved FQN. Set `queryType = "FQN"`.

Path B: If `request.fqn()` is null, try resolving `request.query()` as a class name:
- Call `graphQueryService.searchByName(request.query())`
- Filter results to exact matches: `entry.simpleName().equalsIgnoreCase(request.query())` OR `entry.fullyQualifiedName().equals(request.query())`
- If exactly 1 match: use its FQN. Set `queryType = "SIMPLE_NAME"`.
- If 2+ matches: return a `RagResponse` with `disambiguation` populated and all other fields null. Map matches to `DisambiguationCandidate` records. Set `queryType = "SIMPLE_NAME"`.
- If 0 matches: fall through to natural language path.

Path C: Natural language fallback. Set `queryType = "NATURAL_LANGUAGE"`.
- Embed `request.query()` using `embeddingModel.embed(request.query())`
- Search full Qdrant collection (no cone filter): call `vectorSearchService.searchByCone(queryVector, null, 3)` — but since searchByCone returns empty for null/empty list, instead use `vectorSearchService.search(new SearchRequest(request.query(), 3, request.module(), request.stereotype(), null))`
- Take top 3 results' `classFqn` values
- For each, run `findDependencyConeWithHops()` (the private method below)
- Merge all 3 cone maps: for overlapping FQNs, take minimum hop distance
- Use merged cone for vector search. For graph proximity, all classes get `proximity = 1.0 / max(hopDist, 1)` as normal.
- Set the first hit's chunk data as the "focal class" detail (or build a synthetic focal class from the query)

**Step 2 — Hop-Distance Cone Expansion (for Paths A and B):**

Create a private method `findDependencyConeWithHops(String fqn) -> Map<String, Integer>` that executes this Cypher via `neo4jClient`:
```cypher
MATCH (focal:JavaClass {fullyQualifiedName: $fqn})
OPTIONAL MATCH path = (focal)-[:DEPENDS_ON|EXTENDS|IMPLEMENTS|CALLS|BINDS_TO|QUERIES|MAPS_TO_TABLE*1..10]->(reachable:JavaClass)
WITH focal, reachable, min(length(path)) AS hopDist
WHERE reachable IS NOT NULL
RETURN reachable.fullyQualifiedName AS fqn, hopDist
```
Parse results into `Map<String, Integer>` (FQN to hop distance). Always add the focal class itself at hop 0: `result.put(fqn, 0)`.

**CRITICAL Cypher note (from Phase 7 memory):** `min(length(path))` is an aggregation function. The WITH clause `WITH focal, reachable, min(length(path)) AS hopDist` correctly groups by `(focal, reachable)` and computes minimum path length per reachable node. Do NOT add extra non-aggregated variables to the WITH clause.

**Step 3 — Parallel Graph + Embed Execution (for Paths A and B):**

Use `CompletableFuture.supplyAsync()` to parallelize two independent operations:
```java
long startMs = System.currentTimeMillis();

// Fetch focal class metadata from Neo4j (needed for embedding + response)
// Use a Cypher query: MATCH (c:JavaClass {fullyQualifiedName: $fqn}) RETURN c.simpleName, c.packageName, c.sourceFilePath, c.enhancedRiskScore, c.structuralRiskScore, labels(c)
// Extract stereotype from labels, risk scores, etc.

CompletableFuture<Map<String, Integer>> coneFuture =
    CompletableFuture.supplyAsync(() -> findDependencyConeWithHops(resolvedFqn));

// For the embedding: read the focal class's source file from disk (sourceFilePath from Neo4j)
// If file is not accessible, fall back to simpleName + stereotype as embedding text
String embeddingText = readFocalClassText(focalMetadata);
CompletableFuture<float[]> embeddingFuture =
    CompletableFuture.supplyAsync(() -> embeddingModel.embed(embeddingText));

Map<String, Integer> coneWithHops = coneFuture.join();
float[] queryVector = embeddingFuture.join();
```

**Step 4 — Cone-Constrained Qdrant Search:**

```java
List<String> coneFqns = new ArrayList<>(coneWithHops.keySet());
int limit = request.limit() != null ? Math.min(request.limit(), 100) : 20;
// Request 3x limit from Qdrant to have enough for re-ranking
List<ChunkSearchResult> vectorResults = vectorSearchService.searchByCone(queryVector, coneFqns, limit * 3);
```

**Step 5 — Merge and Re-Rank:**

For each `ChunkSearchResult`:
1. Look up `hopDist` from `coneWithHops.get(result.classFqn())`. The focal class itself has hopDist=0.
2. Compute `graphProximity = 1.0 / Math.max(hopDist, 1)`. For hop 0 (focal class): proximity = 1.0.
3. Compute `finalScore = ragWeightConfig.getVectorSimilarity() * result.score() + ragWeightConfig.getGraphProximity() * graphProximity + ragWeightConfig.getRiskScore() * result.enhancedRiskScore()`
4. Build `ScoreBreakdown(result.score(), graphProximity, result.enhancedRiskScore(), finalScore)`
5. Build `ContextChunk` from the `ChunkSearchResult` fields + score breakdown + relationship path string

**Relationship path string:** For hop 0: `"focal class"`. For hop 1, query the direct relationship type from Neo4j (optional optimization — simpler approach: just use `"(N hop[s])"` format). For 2+ hops: `"multi-hop (N hops)"`. Use Claude's discretion for exact format.

6. Sort all `ContextChunk` by `finalScore` descending
7. Take top `limit` results

**Step 6 — Build ConeSummary:**

Iterate over `vectorResults` (all of them, not just top-limit):
- `totalNodes` = `coneWithHops.size()`
- `vaadin7Count` = count of distinct classFqns where `vaadin7Detected == true`
- `avgEnhancedRisk` = average of `enhancedRiskScore` across all cone chunks
- `topDomainTerms`: Parse `domainTerms` JSON string from each chunk (format: `[{"termId":"t1","displayName":"Invoice"}]`), aggregate by termId, count occurrences, take top 10 by frequency. Use Jackson ObjectMapper for JSON parsing.
- `uniqueBusinessTermCount` = count of distinct termIds across all chunks

**Step 7 — Build FocalClassDetail:**

Use the focal class metadata from Step 3. For `codeText`:
- If `request.includeFullSource()`: read the `.java` file from `sourceFilePath` from Neo4j
- Otherwise: use a summary string like `"[CLASS_HEADER] {simpleName} ({stereotype}) - {packageName}"` or the first CLASS_HEADER chunk text if the focal class appears in vector results

**Step 8 — Build and return RagResponse:**

```java
long durationMs = System.currentTimeMillis() - startMs;
return new RagResponse(queryType, focalClassDetail, contextChunks, coneSummary, null, durationMs);
```

**Private helper: `readFocalClassText(Map<String, Object> focalMetadata) -> String`**
Try to read source file. If unavailable, construct from: `simpleName + " " + stereotype + " " + packageName`. This gives the embedding model enough semantic signal.

**RagController.java** — `com.esmp.rag.api.RagController`:

```java
@RestController
@RequestMapping("/api/rag")
public class RagController {
    private final RagService ragService;

    public RagController(RagService ragService) { this.ragService = ragService; }

    @PostMapping("/context")
    public ResponseEntity<?> getContext(@RequestBody RagRequest request) {
        // Inline validation (same pattern as VectorSearchController)
        if (request == null) {
            return ResponseEntity.badRequest().body(Map.of("error", "Request body is required"));
        }
        if ((request.query() == null || request.query().isBlank()) && (request.fqn() == null || request.fqn().isBlank())) {
            return ResponseEntity.badRequest().body(Map.of("error", "Either 'query' or 'fqn' must be provided"));
        }
        if (request.limit() != null && (request.limit() < 1 || request.limit() > 100)) {
            return ResponseEntity.badRequest().body(Map.of("error", "Limit must be between 1 and 100"));
        }

        RagResponse response = ragService.assemble(request);

        if (response.isDisambiguation()) {
            return ResponseEntity.ok(response);
        }
        if (response.focalClass() == null) {
            return ResponseEntity.status(404).body(Map.of("error", "No class found for query: " + request.query()));
        }
        return ResponseEntity.ok(response);
    }
}
```

Use `java.util.Map` for error response bodies. Add `@RequestMapping("/api/rag")` at class level. Import `org.springframework.http.ResponseEntity`.

**RagValidationQueryRegistry.java** — `com.esmp.rag.validation.RagValidationQueryRegistry`:

Follow the exact pattern from `PilotValidationQueryRegistry` or `LexiconValidationQueryRegistry`. Extend `ValidationQueryRegistry` with 3 queries:

1. `RAG_CONE_QUERY_FUNCTIONAL` — verifies that the cone Cypher returns non-empty results for classes with known dependencies:
```cypher
MATCH (c:JavaClass)-[:DEPENDS_ON|EXTENDS|IMPLEMENTS|CALLS*1..3]->(target:JavaClass)
WITH c, count(DISTINCT target) AS reachable
WHERE reachable > 0
RETURN 'RAG_CONE_QUERY_FUNCTIONAL' AS queryName,
       CASE WHEN count(c) > 0 THEN 'PASS' ELSE 'FAIL' END AS status,
       'Classes with reachable dependencies exist for cone queries' AS description,
       count(c) AS detail
```

2. `RAG_VECTOR_INDEX_ALIGNED` — verifies that classes in the graph have corresponding Qdrant chunks (validated by checking that at least one class with DEPENDS_ON edges also has a sourceFilePath):
```cypher
MATCH (c:JavaClass)
WHERE c.sourceFilePath IS NOT NULL AND c.sourceFilePath <> ''
WITH count(c) AS indexedClasses
RETURN 'RAG_VECTOR_INDEX_ALIGNED' AS queryName,
       CASE WHEN indexedClasses > 0 THEN 'PASS' ELSE 'FAIL' END AS status,
       'Classes with source paths available for vector indexing' AS description,
       indexedClasses AS detail
```

3. `RAG_RISK_SCORES_AVAILABLE` — verifies that enhanced risk scores exist for ranking:
```cypher
MATCH (c:JavaClass)
WHERE c.enhancedRiskScore IS NOT NULL AND c.enhancedRiskScore > 0
WITH count(c) AS scoredClasses
RETURN 'RAG_RISK_SCORES_AVAILABLE' AS queryName,
       CASE WHEN scoredClasses > 0 THEN 'PASS' ELSE 'FAIL' END AS status,
       'Classes with enhanced risk scores for RAG ranking' AS description,
       scoredClasses AS detail
```
