Add a new public method `searchByCone` to `VectorSearchService` that accepts a pre-computed float[] query vector and a list of cone FQNs, and searches Qdrant filtered to those FQNs only.

**New method signature:**
```java
public List<ChunkSearchResult> searchByCone(float[] queryVector, List<String> coneFqns, int limit)
```

**Implementation details:**
1. Add import: `import static io.qdrant.client.ConditionFactory.matchKeywords;` (note: plural — `matchKeywords` not `matchKeyword`)
2. Guard: if `coneFqns` is null or empty, return `List.of()` immediately (empty cone = no results)
3. Build `Filter.Builder` with `filterBuilder.addMust(matchKeywords("classFqn", coneFqns))` — this produces a Qdrant `MatchKeywords` condition that filters points where the `classFqn` payload field is in the provided list. The `classFqn` payload index was created in Phase 8 by `QdrantCollectionInitializer`.
4. Build `SearchPoints.Builder`:
   - `.setCollectionName(vectorConfig.getCollectionName())`
   - `.setLimit(limit)`
   - `.setWithPayload(enable(true))`
   - `.setFilter(filterBuilder.build())`
   - Iterate `for (float v : queryVector) { builder.addVector(v); }` — same pattern as existing `search()` method
5. Execute: `qdrantClient.searchAsync(builder.build()).get(30, TimeUnit.SECONDS)`
6. Map results using the existing private `mapToResult(ScoredPoint)` method
7. Return `List<ChunkSearchResult>`
8. Wrap the async call in try/catch, log error and rethrow as RuntimeException — same pattern as existing `search()` method

This method differs from the existing `search()` in two ways:
- It accepts a pre-computed `float[]` vector (the caller embeds the text separately for parallelism)
- It uses `matchKeywords` (list filter) instead of individual `matchKeyword` (single value filter)

Do NOT modify the existing `search(SearchRequest)` method. The new method is additive.

Add Javadoc:
```java
/**
 * Searches Qdrant for chunks whose classFqn is in the provided cone FQN list.

 *
 * @param queryVector pre-computed embedding vector (384 dimensions)

 * @param limit maximum number of results
 * @return ranked chunk results within the cone boundary
 */
```
