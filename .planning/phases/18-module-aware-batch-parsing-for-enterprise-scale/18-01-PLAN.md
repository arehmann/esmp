<objective>
Create the module detection subsystem that parses Gradle and Maven build files, builds the inter-module dependency graph, and produces topologically sorted parse waves.

Purpose: This is the foundation for module-aware extraction. Without accurate module detection and ordering, the per-module classpath strategy cannot work because dependent modules need upstream modules parsed first.

Output: ModuleDetectionService, ModuleGraph, supporting records, unit tests, and test fixtures.
</objective>

    3. Create `ModuleDetectionResult` record in `src/main/java/com/esmp/extraction/module/ModuleDetectionResult.java`:
       ```java
       public record ModuleDetectionResult(
           BuildSystem buildSystem,
           List<List<ModuleDescriptor>> waves,  // topologically sorted waves of modules
           List<SkippedModule> skippedModules,
           int totalModules,
           int totalJavaFiles
       ) {
           public record SkippedModule(String name, String reason) {}
           public boolean isMultiModule() { return buildSystem != BuildSystem.NONE && !waves.isEmpty(); }
       }
       ```

    4. Create `ModuleGraph` in `src/main/java/com/esmp/extraction/module/ModuleGraph.java`:
       - Constructor takes `Map<String, ModuleDescriptor> modules` (name -> descriptor)
       - `computeWaves()` method implements Kahn's BFS algorithm (same pattern as SchedulingService):
         - Build inDegree map from each module's dependsOn list
         - While inDegree has entries: collect modules with inDegree==0 as a wave, remove them, decrement dependents' inDegree
         - If a wave is empty but modules remain: cycle detected, add all remaining as final wave
       - Returns `List<List<ModuleDescriptor>>` — each inner list is a parallel wave
       - Stores `skippedModules` list separately from waves

    5. Create `ModuleDetectionService` as `@Service` in `src/main/java/com/esmp/extraction/module/ModuleDetectionService.java`:
       - `detect(Path sourceRoot)` method:
         a. Check for `settings.gradle` at sourceRoot -> if exists, call `detectGradle(sourceRoot)`
         b. Else check for `pom.xml` at sourceRoot with `<modules>` element -> if exists, call `detectMaven(sourceRoot)`
         c. Else return `new ModuleDetectionResult(BuildSystem.NONE, List.of(), List.of(), 0, 0)`
       - `detectGradle(Path sourceRoot)`:
         a. Parse `settings.gradle` using regex `Pattern.compile("['\"][:.]?([^'\":/]+)['\"]")` on include lines
         b. For each module name, resolve `sourceRoot.resolve(moduleName)` as module directory
         c. Check `moduleDir.resolve("src/main/java")` exists — skip module if not, add to skippedModules
         d. Check `moduleDir.resolve("build/classes/java/main")` exists — skip module if not, reason "compiled classes not found at build/classes/java/main"
         e. Parse `moduleDir.resolve("build.gradle")` for `project\s*\(\s*['"][:.]?([^'"]+)['"]\s*\)` to extract inter-module dependencies
         f. Only include dependencies that are in the detected module set (ignore external project references)
         g. Scan `sourceDir` for .java files using `Files.walk().filter(p -> p.toString().endsWith(".java"))`
         h. Build ModuleDescriptor for each valid module
         i. Construct ModuleGraph, call computeWaves(), return ModuleDetectionResult
       - `detectMaven(Path sourceRoot)`:
         a. Parse root `pom.xml` using `javax.xml.parsers.DocumentBuilderFactory` — extract `<modules>/<module>` text content
         b. For each module, resolve directory and check `src/main/java` exists
         c. Check `target/classes` directory exists — skip if not
         d. Parse each module's `pom.xml` for `<dependencies>/<dependency>` where `<groupId>` equals `${project.groupId}` or the parent's groupId — extract `<artifactId>` as inter-module dep
         e. Filter deps to only include those in detected module set
         f. Build ModuleGraph and return result
