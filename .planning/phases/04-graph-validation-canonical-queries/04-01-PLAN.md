Relationship directions (from LinkingService.java — canonical source):
- `(pkg)-[:CONTAINS_CLASS]->(c)` — package to class
- `(mod)-[:CONTAINS_PACKAGE]->(pkg)` — module to package
- `(c)-[:EXTENDS]->(parent)` — class to superclass
- `(c)-[:IMPLEMENTS]->(iface)` — class to interface
- `(c)-[:DEPENDS_ON]->(dep)` — class to dependency
- `(c)-[:MAPS_TO_TABLE]->(table)` — class to DB table
- `(c)-[:DECLARES_METHOD]->(m)` — class to method
- `(c)-[:DECLARES_FIELD]->(f)` — class to field
- `(c)-[:HAS_ANNOTATION]->(a)` — class to annotation
- `(m)-[:CALLS]->(m2)` — method to method
- `(m)-[:QUERIES]->(table)` — method to DB table
- `(view)-[:BINDS_TO]->(entity)` — view to entity class

Node types and ID properties (from Neo4jSchemaInitializer):
- JavaClass: fullyQualifiedName (unique constraint)
- JavaMethod: methodId (unique constraint)
- JavaField: fieldId (unique constraint)
- JavaAnnotation: fullyQualifiedName (unique constraint)
- JavaPackage: packageName (unique constraint)
- JavaModule: moduleName (unique constraint)
- DBTable: tableName (unique constraint)

**1. ValidationSeverity.java** — enum with ERROR, WARNING.

**3. ValidationQuery.java** — record with fields: `String name`, `String description`, `String cypher`, `ValidationSeverity severity`. Each Cypher query MUST return exactly two columns: `count AS count` (long) and `details AS details` (list of strings, capped at 20 via `[0..20]`).

**4. ValidationQueryRegistry.java** — `@Component` that initializes all 20 queries in its constructor and exposes `List<ValidationQuery> getQueries()` as an unmodifiable list. The 20 queries are:

**Structural integrity (10 queries):**
1. ORPHAN_CLASS_NODES (WARNING): JavaClass nodes with no incoming CONTAINS_CLASS edge from any JavaPackage.
   `OPTIONAL MATCH (c:JavaClass) WHERE NOT ()-[:CONTAINS_CLASS]->(c) RETURN count(c) AS count, collect(c.fullyQualifiedName)[0..20] AS details`
2. DANGLING_METHOD_NODES (ERROR): JavaMethod nodes with no incoming DECLARES_METHOD edge.
   `OPTIONAL MATCH (m:JavaMethod) WHERE NOT ()-[:DECLARES_METHOD]->(m) RETURN count(m) AS count, collect(m.methodId)[0..20] AS details`
3. DANGLING_FIELD_NODES (ERROR): JavaField nodes with no incoming DECLARES_FIELD edge.
   `OPTIONAL MATCH (f:JavaField) WHERE NOT ()-[:DECLARES_FIELD]->(f) RETURN count(f) AS count, collect(f.fieldId)[0..20] AS details`
4. DANGLING_ANNOTATION_NODES (WARNING): JavaAnnotation nodes with no incoming HAS_ANNOTATION edge.
   `OPTIONAL MATCH (a:JavaAnnotation) WHERE NOT ()-[:HAS_ANNOTATION]->(a) RETURN count(a) AS count, collect(a.fullyQualifiedName)[0..20] AS details`
5. DUPLICATE_CLASS_FQNS (ERROR): JavaClass nodes sharing the same fullyQualifiedName (constraint violation check).
   `MATCH (c:JavaClass) WITH c.fullyQualifiedName AS fqn, count(*) AS cnt WHERE cnt > 1 RETURN count(fqn) AS count, collect(fqn)[0..20] AS details`
6. EXTENDS_CHAIN_DANGLING (WARNING): Classes with superClass property set but no EXTENDS outgoing edge.
   `OPTIONAL MATCH (c:JavaClass) WHERE c.superClass IS NOT NULL AND c.superClass <> '' AND NOT (c)-[:EXTENDS]->() RETURN count(c) AS count, collect(c.fullyQualifiedName + ' -> ' + c.superClass)[0..20] AS details`
7. IMPLEMENTS_MISSING (WARNING): Classes with implementedInterfaces containing FQNs that exist as JavaClass nodes in the graph but no IMPLEMENTS edge to them.
   `OPTIONAL MATCH (c:JavaClass) WHERE size(c.implementedInterfaces) > 0 AND NOT (c)-[:IMPLEMENTS]->() RETURN count(c) AS count, collect(c.fullyQualifiedName)[0..20] AS details`
8. MAPS_TO_TABLE_ORPHAN_TABLE (WARNING): DBTable nodes with no incoming MAPS_TO_TABLE edge.
   `OPTIONAL MATCH (t:DBTable) WHERE NOT ()-[:MAPS_TO_TABLE]->(t) RETURN count(t) AS count, collect(t.tableName)[0..20] AS details`
9. QUERIES_EDGE_INTEGRITY (ERROR): QUERIES relationships where source is not JavaMethod or target is not DBTable.
   `OPTIONAL MATCH (src)-[r:QUERIES]->(tgt) WHERE NOT src:JavaMethod OR NOT tgt:DBTable RETURN count(r) AS count, collect(COALESCE(src.methodId, 'unknown') + ' -QUERIES-> ' + COALESCE(tgt.tableName, 'unknown'))[0..20] AS details`
10. BINDS_TO_EDGE_INTEGRITY (ERROR): BINDS_TO relationships where source is not JavaClass or target is not JavaClass.
    `OPTIONAL MATCH (src)-[r:BINDS_TO]->(tgt) WHERE NOT src:JavaClass OR NOT tgt:JavaClass RETURN count(r) AS count, collect(COALESCE(src.fullyQualifiedName, 'unknown') + ' -BINDS_TO-> ' + COALESCE(tgt.fullyQualifiedName, 'unknown'))[0..20] AS details`

**Architectural pattern checks (10 queries):**
11. SERVICE_HAS_DEPENDENCIES (WARNING): Service-labeled classes with zero DEPENDS_ON outgoing edges.
    `OPTIONAL MATCH (c:JavaClass) WHERE ANY(label IN labels(c) WHERE label = 'Service') AND NOT (c)-[:DEPENDS_ON]->() RETURN count(c) AS count, collect(c.fullyQualifiedName)[0..20] AS details`
12. REPOSITORY_HAS_QUERIES (WARNING): Repository-labeled classes whose methods have zero QUERIES outgoing edges.
    `OPTIONAL MATCH (c:JavaClass) WHERE ANY(label IN labels(c) WHERE label = 'Repository') AND NOT (c)-[:DECLARES_METHOD]->()-[:QUERIES]->() RETURN count(c) AS count, collect(c.fullyQualifiedName)[0..20] AS details`
13. UI_VIEW_HAS_BINDS_TO (WARNING): VaadinView-labeled classes with zero BINDS_TO outgoing edges.
    `OPTIONAL MATCH (c:JavaClass) WHERE ANY(label IN labels(c) WHERE label = 'VaadinView') AND NOT (c)-[:BINDS_TO]->() RETURN count(c) AS count, collect(c.fullyQualifiedName)[0..20] AS details`
14. ENTITY_MAPS_TO_TABLE (WARNING): Classes with @Entity annotation (via HAS_ANNOTATION to javax.persistence.Entity or jakarta.persistence.Entity) but no MAPS_TO_TABLE outgoing edge.
    `OPTIONAL MATCH (c:JavaClass)-[:HAS_ANNOTATION]->(a:JavaAnnotation) WHERE (a.fullyQualifiedName = 'javax.persistence.Entity' OR a.fullyQualifiedName = 'jakarta.persistence.Entity') AND NOT (c)-[:MAPS_TO_TABLE]->() RETURN count(c) AS count, collect(c.fullyQualifiedName)[0..20] AS details`
15. INHERITANCE_CHAIN_COMPLETENESS (ERROR): Classes whose superClass FQN exists as a JavaClass node in the graph but the EXTENDS edge is missing (stronger check than #6 — parent is present but edge is not).
    `OPTIONAL MATCH (c:JavaClass), (parent:JavaClass) WHERE c.superClass = parent.fullyQualifiedName AND NOT (c)-[:EXTENDS]->(parent) RETURN count(c) AS count, collect(c.fullyQualifiedName + ' -> ' + c.superClass)[0..20] AS details`
16. TRANSITIVE_REPO_DEPENDENCY (WARNING): Service classes that have no transitive path to any Repository class (max 10 hops).
    `OPTIONAL MATCH (s:JavaClass) WHERE ANY(label IN labels(s) WHERE label = 'Service') AND NOT EXISTS { MATCH (s)-[:DEPENDS_ON*1..10]->(r:JavaClass) WHERE ANY(l IN labels(r) WHERE l = 'Repository') } RETURN count(s) AS count, collect(s.fullyQualifiedName)[0..20] AS details`
17. NO_ISOLATED_MODULES (WARNING): JavaModule nodes with no CONTAINS_PACKAGE outgoing edges.
    `OPTIONAL MATCH (m:JavaModule) WHERE NOT (m)-[:CONTAINS_PACKAGE]->() RETURN count(m) AS count, collect(m.moduleName)[0..20] AS details`
18. NO_EMPTY_PACKAGES (WARNING): JavaPackage nodes with no CONTAINS_CLASS outgoing edges.
    `OPTIONAL MATCH (p:JavaPackage) WHERE NOT (p)-[:CONTAINS_CLASS]->() RETURN count(p) AS count, collect(p.packageName)[0..20] AS details`
19. ANNOTATION_COVERAGE (WARNING): JavaClass nodes with empty annotations list and no HAS_ANNOTATION edge (fully unannotated — informational).
    `OPTIONAL MATCH (c:JavaClass) WHERE (c.annotations IS NULL OR size(c.annotations) = 0) AND NOT (c)-[:HAS_ANNOTATION]->() RETURN count(c) AS count, collect(c.fullyQualifiedName)[0..20] AS details`
20. CALLS_EDGE_COVERAGE (WARNING): Total CALLS edges in graph — sanity check, should be > 0 after extraction. NOTE: For this query the semantics are inverted — count > 0 means PASS, count = 0 means WARN. Handle this specially in the service: if `query.name().equals("CALLS_EDGE_COVERAGE")`, invert the pass/fail logic (count == 0 means warning, count > 0 means pass).
    `OPTIONAL MATCH ()-[r:CALLS]->() RETURN count(r) AS count, [] AS details`

**5. ValidationQueryResult.java** (in `com.esmp.graph.api`) — record: `String name`, `String description`, `ValidationSeverity severity`, `ValidationStatus status`, `long count`, `List<String> details`.

**6. ValidationReport.java** (in `com.esmp.graph.api`) — record: `String generatedAt` (ISO-8601), `List<ValidationQueryResult> results`, `long errorCount`, `long warnCount`, `long passCount`.

**7. ValidationService.java** — `@Service` with `Neo4jClient` and `List<ValidationQueryRegistry>` injected (list injection for extensibility — future phases add their own registry beans). Method `runAllValidations()`:
- Collects all queries from all registries
- Executes each via `neo4jClient.query(cypher).fetch().all()`
- Each query returns 1 row with `count` (Long) and `details` (List). Extract using `Map<String, Object>` pattern.
- Determines status: count == 0 => PASS (except CALLS_EDGE_COVERAGE where count > 0 => PASS). For non-zero count: ERROR severity => FAIL, WARNING severity => WARN.
- Returns ValidationReport with timestamp, all results, errorCount, warnCount, passCount.

IMPORTANT: Use `neo4jClient.query(cypher).fetch().all()` returning `Collection<Map<String, Object>>` — NOT `fetchAs()`. The map-based access handles mixed count+list return shapes cleanly. When extracting `details` from the map, handle the Neo4j driver Value type: if the value is a `List<?>`, stream and filter to `List<String>`. See RESEARCH.md Pitfall 4 for the pattern.

**8. ValidationController.java** (in `com.esmp.graph.api`) — `@RestController` with `@RequestMapping("/api/graph")`. Single endpoint:
- `@GetMapping("/validation")` — calls `validationService.runAllValidations()`, returns `ResponseEntity<ValidationReport>` with 200 status.
