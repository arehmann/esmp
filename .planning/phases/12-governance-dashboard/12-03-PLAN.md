Constructor injects `DashboardService`. The constructor calls private builder methods for each section and adds them to the layout. Data is loaded ONCE in the constructor (no polling, no auto-refresh per user decision).

**Section 1: Metric Summary Cards (DASH-01, DASH-05, DASH-06)**

Create a `HorizontalLayout` with 3 cards using a private `buildMetricCard(String title, String value, String subtitle, String sectionId)` helper:

Card 1 — "Vaadin 7 APIs":
- Value: compute overall V7 % from `moduleSummaries` as `sum(vaadin7Count) / sum(classCount) * 100`, format as `"%.1f%%"`.
- Subtitle: `"X modules with V7 usage"` where X = count of modules with vaadin7Count > 0.
- sectionId: `"v7-details"`

Card 2 — "Lexicon Coverage":
- Value: `lexiconCoverage.coveragePct()` formatted as `"%.1f%%"`.
- Subtitle: `"%d / %d terms curated"` using curated and total.
- sectionId: `"lexicon-details"` (but scrolls to existing Lexicon view — use `UI.getCurrent().navigate("lexicon")` on click instead)

Card 3 — "Migration Progress":
- Value: compute overall migration progress as `(1 - overallV7Pct) * 100` formatted as `"%.1f%%"`.
- Subtitle: `"%d modules analyzed"`.
- sectionId: `"heatmap-section"`

Each card is a `Div` styled with border, border-radius, padding, cursor:pointer. On click, `getElement().executeJs("document.getElementById($0).scrollIntoView({behavior:'smooth'})", sectionId)` — EXCEPT Card 2 which navigates to `/lexicon`.

**Mini module breakdown below cards:**

After the card row, add a `Details` component (or `Div` with toggle) showing a `Grid<ModuleSummary>` with columns: Module, Classes, V7 Count, V7 %, Avg Risk, High Risk Count. Use `ListDataProvider<ModuleSummary>`. This fulfills the "mini module-level breakdown inline" requirement.

**Section 2: Migration Heatmap (DASH-06)**

Add an `H3("Migration Heatmap")` with `id="heatmap-section"`.

Create a `Grid<ModuleSummary>` sorted by `heatmapScore` descending. Columns: Module, V7 %, Avg Risk, Heatmap Score. Use `ComponentRenderer` for the heatmap score column to render a colored `Span` where:
- Green (`#22c55e`): heatmapScore < 0.1
- Yellow (`#eab308`): heatmapScore 0.1 - 0.3
- Orange (`#f97316`): heatmapScore 0.3 - 0.5
- Red (`#ef4444`): heatmapScore >= 0.5

The color logic: `heatmapScore = vaadin7Pct * avgEnhancedRisk`. High V7% + high risk = most red.

**Section 3: Risk Hotspot Clusters (DASH-04)**

Add an `H3("Risk Hotspot Clusters")` with `id="risk-clusters"`.

Create a `CytoscapeGraph` instance for the risk cluster visualization. Convert `List<RiskCluster>` to Cytoscape.js elements JSON:
- Each module = one node. `id` = module name, `label` = module name, `type` = "module".
- `size` = `Math.max(20, Math.min(80, classCount * 3))` (proportional to class count, clamped 20-80).
- `color` = risk-based: avgRisk < 0.3 -> "#22c55e" (green), 0.3-0.6 -> "#eab308" (yellow), >= 0.6 -> "#ef4444" (red).
- No edges in this view (it's a bubble map, not a graph). Use layout `{ name: 'circle' }` override or `cose` default.

Build JSON using `com.fasterxml.jackson.databind.ObjectMapper` (already on classpath via Spring Boot) to serialize a `List<Map<String, Object>>` of Cytoscape element maps:
```java
List<Map<String, Object>> elements = new ArrayList<>();
for (RiskCluster rc : riskClusters) {
    elements.add(Map.of(
        "group", "nodes",
        "data", Map.of(
            "id", rc.module(),
            "label", rc.module() + "\n" + rc.classCount() + " classes",
            "type", "riskCluster",
            "size", Math.max(20, Math.min(80, rc.classCount() * 3)),
            "color", riskColor(rc.avgRisk())
        )
    ));
}
String json = objectMapper.writeValueAsString(elements);
riskClusterGraph.setGraphData(json);
```

On node click, show a `Notification` or update a side `Div` with the cluster details (module, classCount, avgRisk, maxRisk, highRiskCount).

**Section 4: Dependency Graph Explorer (DASH-02)**

Add an `H3("Dependency Graph")` with `id="dependency-graph"`.

Create a `HorizontalLayout` with:
- Left (75% width): `CytoscapeGraph` for the dependency graph.
- Right (25% width): A `Div` side panel (initially showing "Click a node to see details").

Load module-level overview from `dashboardService.getModuleDependencyEdges()`:
- Create a unique set of module names from all edges (both source and target).
- Each module = a node: `id` = module name, `label` = module name, `type` = "module", `color` = "#3b82f6", `size` = 40.
- Each edge: `source` = edge.source(), `target` = edge.target(), `width` = `Math.max(1, Math.min(5, edge.weight() / 10))`.

Serialize to Cytoscape elements JSON and call `setGraphData(json)`.

On `NodeClickEvent`:
- If `nodeType` is `"module"`: call `dashboardService.getClassesInModule(nodeId)`, rebuild graph with class-level nodes WITHIN that module (replacing the module-level view). Each class node: `id` = fqn, `label` = simpleName, `type` = "class", `color` based on riskScore, `size` = 30. Edges from `dependsOn` list. Update side panel with module name.
- If `nodeType` is `"class"`: update the side panel `Div` to show: FQN, risk score, labels (stereotypes), dependencies. Format as:
  ```
  H4: simpleName
  Span: fqn
  Span: Risk: X.XX
  Span: Labels: [Service, VaadinView, ...]
  Span: Depends on: [list of FQNs]
  ```

Add a `Button("Back to modules")` visible only when in class-level view, which reloads the module-level graph.

**Section 5: Business Concept Graph (DASH-03)**

Add an `H3("Business Concept Graph")` with `id="concept-graph"`.

Create a `HorizontalLayout` with:
- Left (75% width): `CytoscapeGraph`.
- Right (25% width): Side panel `Div`.

Load data from `dashboardService.getBusinessTermGraph()`:
- Each `BusinessTermSummary` = a node: `id` = termId, `label` = displayName, `type` = "term", `color` = criticality-based (High="#ef4444", Medium="#eab308", Low="#22c55e"), `size` = `Math.max(20, Math.min(60, classFqns.size() * 10))`.
- For each term, each class FQN in `classFqns` = a node: `id` = fqn, `label` = last segment of FQN (simple name), `type` = "class", `color` = "#6366f1", `size` = 25.
- Edge from each class to its term: `source` = fqn, `target` = termId, `width` = 1.
- Deduplicate class nodes (a class may link to multiple terms).

On `NodeClickEvent`: update side panel with term details (displayName, criticality, curated status, linked classes) or class details (FQN, linked terms).

**General styling:**
- Each section separated by `Hr()` or margin.
- Cards row uses `FlexLayout` or `HorizontalLayout` with `setFlexWrap(FlexLayout.FlexWrap.WRAP)` for responsive layout.
- Use Lumo utility classes where possible: `addClassNames("p-m")` for padding.
- `setPadding(true)` and `setSpacing(true)` on the outer VerticalLayout.
- Graph sections have a minimum height of 500px.

**Error handling:**
- Wrap each data load in try/catch. On error, show a `Span("Failed to load [section name]")` with red text.
- If `moduleSummaries` is empty, show "No modules found — run extraction first" in the cards area.

**ObjectMapper:**
- Inject or create `new ObjectMapper()` as a field. Used to serialize Cytoscape element lists to JSON strings.
