1. **MigrationContextAssembler.java** — Create `src/main/java/com/esmp/mcp/application/MigrationContextAssembler.java`:
   - `@Service`, NOT `@Transactional` (pure read orchestrator like RagService, SchedulingService)
   - Constructor-inject: `GraphQueryService`, `RiskService`, `LexiconService`, `VectorSearchService`, `EmbeddingModel` (from Spring AI), `McpConfig`, `Neo4jClient`
   - **`assemble(String classFqn)`** method:
     - Record `startMs = System.currentTimeMillis()`
     - Fire 4 parallel `CompletableFuture.supplyAsync()` calls:
       1. **Cone**: `graphQueryService.findDependencyCone(classFqn)` — returns `Optional<DependencyConeResponse>`
       2. **Risk**: `riskService.getClassDetail(classFqn)` — returns `Optional<RiskDetailResponse>`
       3. **Terms**: Find domain terms for this class via Neo4jClient Cypher: `MATCH (c:JavaClass {fullyQualifiedName: $fqn})-[:USES_TERM]->(t:BusinessTerm) RETURN t.termId, t.displayName, t.definition, t.criticality, t.migrationSensitivity, t.synonyms, t.status, t.usageCount` — map to `List<BusinessTermResponse>`
       4. **Rules**: Find business rules via Neo4jClient Cypher: `MATCH (c:JavaClass {fullyQualifiedName: $fqn})-[:DEFINES_RULE]->(rule:BusinessTerm) RETURN rule.displayName` — returns `List<String>`
     - Join each future with try-catch wrapping. On failure:
       - Add `AssemblerWarning(serviceName, exceptionMessage)` to warnings list
       - Use empty/null fallback for that section
       - Increment completeness contributions only for successful services
     - **Completeness scoring**: cone=0.30, risk=0.25, terms=0.20, rules=0.10, codeChunks=0.15. Sum of successful weights = `contextCompleteness`
     - **Code chunks**: After cone resolves successfully, embed `classFqn` via `embeddingModel.embed(classFqn)` and call `vectorSearchService.searchByCone(embedding, coneFqns, 20)` where `coneFqns` is extracted from `coneResponse.coneNodes()` FQNs. If cone failed, skip code chunks. Wrap in try-catch with warning on failure.
     - **Token budget truncation**: After all data assembled:
       - Estimate tokens: `estimateTokens(object)` = serialize to JSON string length / 4 (conservative chars-per-token for mixed code/English)
       - Priority order (highest first): businessRules, domainTerms, riskAnalysis, dependencyCone, codeChunks
       - Calculate total tokens. If over `mcpConfig.getContext().getMaxTokens() * 0.9` (90% safety factor):
         - Truncate `codeChunks` list from the end until under budget
         - Set `truncated = true`, `truncatedItems = originalSize - remainingSize`
       - If still over budget after removing all code chunks, truncate dependencyCone.coneNodes (keep first 50)
     - Return `MigrationContext(classFqn, cone, risk, terms, rules, codeChunks, truncated, truncatedItems, completeness, warnings, durationMs)`
   - **`estimateTokens(Object obj)`** private helper: `new ObjectMapper().writeValueAsString(obj).length() / 4`
