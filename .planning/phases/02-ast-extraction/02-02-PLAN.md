  <action>
    1. **ClassMetadataVisitor.java** — extends `JavaIsoVisitor<ExtractionAccumulator>`:
       - `visitClassDeclaration`: Extract FQN via `cd.getType().getFullyQualifiedName()`, simpleName, package, modifiers, annotations (from `getLeadingAnnotations()`), superclass (from `getExtends()`), interfaces (from `getImplements()`), isInterface/isAbstract/isEnum flags. Call `acc.addClass(...)`.
       - `visitMethodDeclaration`: Extract method name, return type, parameter types, annotations, modifiers, isConstructor flag. Build methodId as `declaringClass#methodName(ParamType1,ParamType2)`. Call `acc.addMethod(...)`.
       - `visitVariableDeclarations`: Only for class-level fields (check `getCursor().firstEnclosing(J.MethodDeclaration.class) == null` to exclude local variables). Extract field name, type, annotations, modifiers. Build fieldId as `declaringClass#fieldName`. Call `acc.addField(...)`.
       - CRITICAL: Always call `super.visit*()` to ensure recursion into nested classes per RESEARCH.md anti-pattern warning.

    2. **CallGraphVisitor.java** — extends `JavaIsoVisitor<ExtractionAccumulator>`:
       - `visitMethodInvocation`: Check `mi.getMethodType() != null` (skip if type unresolved — per RESEARCH.md Pitfall 1). Get callee class FQN from `mi.getMethodType().getDeclaringType().getFullyQualifiedName()`. Get callee method name from `mi.getMethodType().getName()`. Walk cursor up to find enclosing `J.MethodDeclaration` for caller context. Build caller/callee methodIds and call `acc.addCall(callerMethodId, calleeMethodId, sourceFile, lineNumber)`.
       - Filter: Skip calls to `java.lang.*` and `java.util.*` standard library to reduce noise. Only record calls where both caller and callee declaring types are in the project's source set (or at minimum, not JDK classes).

    3. **VaadinPatternVisitor.java** — extends `JavaIsoVisitor<ExtractionAccumulator>`:
       - Known Vaadin 7 type sets (constants):
         - `VAADIN_VIEW_TYPES = {"com.vaadin.navigator.View", "com.vaadin.ui.UI"}` — classes implementing/extending these get VaadinView label
         - `VAADIN_COMPONENT_PACKAGE = "com.vaadin.ui"` — any class using types from this package gets VaadinComponent label
         - `VAADIN_DATA_BINDING_TYPES = {"com.vaadin.data.fieldgroup.BeanFieldGroup", "com.vaadin.data.fieldgroup.FieldGroup", "com.vaadin.data.util.BeanItemContainer"}` — classes using these get VaadinDataBinding label
       - `visitClassDeclaration`: Check `getImplements()` and `getExtends()` against view types. Mark accordingly.
       - `visitMethodInvocation`: Detect `addComponent()`/`addComponents()` calls where declaring type starts with `com.vaadin.ui` — record CONTAINS_COMPONENT edges via `acc.addComponentEdge(...)`. Also detect data binding calls (fieldGroup.bind, etc.) and mark class as VaadinDataBinding.
       - `visitNewClass`: Detect `new` expressions for Vaadin component types (e.g., `new Button()`, `new VerticalLayout()`) to mark classes as VaadinComponent users.
