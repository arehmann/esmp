  <action>
    1. **Neo4j Repositories** — Spring Data Neo4j repositories:
       - `ClassNodeRepository extends Neo4jRepository<ClassNode, String>` with custom queries:
         - `long countByFullyQualifiedName(String fqn)` — for duplicate checking
         - `@Query("MATCH (c:JavaClass) RETURN count(c)") long countAll()` — total class count
       - `MethodNodeRepository extends Neo4jRepository<MethodNode, String>`
       - `FieldNodeRepository extends Neo4jRepository<FieldNode, String>`

    2. **AccumulatorToModelMapper.java** — `@Component`:
       - `List<ClassNode> mapToClassNodes(ExtractionAccumulator acc)` — converts accumulator data to @Node entities:
         - For each ClassNodeData: create ClassNode, set all properties, set extraLabels from acc.vaadinViews/vaadinComponents/vaadinDataBindings
         - For each class: create child MethodNode list from acc.methods filtered by declaringClass
         - For each class: create child FieldNode list from acc.fields filtered by declaringClass
         - For each method: create CallsRelationship list from acc.callEdges filtered by callerMethodId
         - For each class in acc.componentEdges: create ContainsComponentRelationship list
       - Map content hash (SHA-256 of source file content) into ClassNode.contentHash for future change detection

    3. **ExtractionService.java** — `@Service`:
       - Injects: `JavaSourceParser`, `AccumulatorToModelMapper`, `ClassNodeRepository`, `VaadinAuditService`, `ExtractionConfig`
       - `ExtractionResult extract(String sourceRoot, String classpathFile)`:
         a. Resolve sourceRoot (use parameter if provided, else fall back to ExtractionConfig.sourceRoot)
         b. Scan sourceRoot recursively for `*.java` files
         c. Call `javaSourceParser.parse(javaPaths, sourceRootPath, classpathFile)`
         d. Create new `ExtractionAccumulator`
         e. For each SourceFile: run ClassMetadataVisitor, CallGraphVisitor, VaadinPatternVisitor
         f. Call `mapper.mapToClassNodes(accumulator)` to get entity list
         g. Save all ClassNode entities via `classNodeRepository.saveAll(classNodes)` — SDN handles MERGE due to @Id + @Version
         h. Generate Vaadin audit report via `vaadinAuditService.generateReport(accumulator)`
         i. Return ExtractionResult with counts and audit report
       - Idempotency: @Id business keys + @Version ensure `saveAll()` merges existing nodes. Neo4j uniqueness constraints from Plan 01 act as safety net.
       - Error handling: Catch exceptions per-file (not per-batch), log file path + error, continue with remaining files. Return partial results with error count in response.

    4. **ExtractionController.java** — `@RestController @RequestMapping("/api/extraction")`:
       - `@PostMapping("/trigger") ResponseEntity<ExtractionResponse> trigger(@RequestBody ExtractionRequest request)`:
         - Request body: `{ "sourceRoot": "/path/to/source", "classpathFile": "/path/to/classpath.txt" }` — both optional (fall back to config)
         - Validates sourceRoot exists and is a directory
         - Calls `extractionService.extract(sourceRoot, classpathFile)`
         - Returns 200 with ExtractionResponse (synchronous per user decision)
         - Returns 400 if sourceRoot is invalid

    5. **ExtractionRequest.java** — record:
       - `String sourceRoot` (optional — overrides config default per user decision)
       - `String classpathFile` (optional)
       - `String moduleFilter` (optional — narrows scope per user decision, filters to subdirectory)

    6. **ExtractionResponse.java** — record:
       - `int classCount`, `int methodCount`, `int fieldCount`, `int callEdgeCount`
       - `int vaadinViewCount`, `int vaadinComponentCount`, `int vaadinDataBindingCount`
       - `int errorCount`, `List<String> errors`
       - `VaadinAuditReport auditReport`
       - `long durationMs`

    7. **VaadinAuditReport.java** — record:
       - `List<PatternEntry> detectedPatterns` — each with name, count, example FQNs
       - `List<String> knownLimitations` — static analysis limitations (conditional addComponent, reflection, etc.)
       - `String summary` — human-readable summary

    8. **VaadinAuditService.java** — `@Service`:
       - `VaadinAuditReport generateReport(ExtractionAccumulator acc)`:
         - Count VaadinView implementations found
         - Count VaadinComponent usages found
         - Count VaadinDataBinding usages found
         - Count CONTAINS_COMPONENT edges found
         - Document known limitations (per RESEARCH.md Pitfall 5): conditional component trees, reflective calls, runtime-only push/widgetset config
         - This addresses the STATE.md blocker: "OpenRewrite Vaadin 7 recipe coverage is LOW confidence — hands-on audit required in Phase 2"
         - The audit report proves what OpenRewrite CAN parse from Vaadin 7 code

    9. **Integration tests** — ExtractionIntegrationTest.java:
       - `@SpringBootTest(webEnvironment = RANDOM_PORT)` with Testcontainers Neo4j (follow Phase 1 pattern with @DynamicPropertySource)
       - Also need MySQL and Qdrant testcontainers (since Spring Boot startup requires all data stores healthy via DataStoreStartupValidator)
       - Use `TestRestTemplate` to POST to `/api/extraction/trigger` with sourceRoot pointing to test fixtures
       - For classpath: create a temporary classpath file containing the Vaadin 7 JAR path resolved from test classpath
       - Assertions per the behavior list above: node counts, relationship existence, dynamic labels, idempotency
       - Neo4j queries to verify: `MATCH (c:JavaClass) RETURN count(c)`, `MATCH (c:JavaClass)-[:DECLARES_METHOD]->(m) RETURN c.simpleName, count(m)`, `MATCH (m1:JavaMethod)-[:CALLS]->(m2:JavaMethod) RETURN m1.methodId, m2.methodId`, `MATCH (c:VaadinView) RETURN c.fullyQualifiedName`
