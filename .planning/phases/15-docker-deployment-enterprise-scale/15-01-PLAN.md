2. **SourceAccessService.java** (`src/main/java/com/esmp/source/application/SourceAccessService.java`):
   - `@Service` class injecting `SourceAccessConfig`
   - Implements `ApplicationListener<ApplicationReadyEvent>` to resolve source on startup
   - Private field `private volatile String resolvedSourceRoot;`
   - `onApplicationEvent(ApplicationReadyEvent)`: calls `resolveSourceRoot()`, logs result. If resolution fails (IOException), log error but do not crash the app — set resolvedSourceRoot to empty string with warning.
   - `public String getResolvedSourceRoot()`: returns the resolved path (may be empty if not configured)
   - `public String resolveSourceRoot()`: switch on `config.getStrategy()`:
     - `VOLUME_MOUNT`: validate `config.getVolumeMountPath()` exists as a directory via `Files.isDirectory(Path.of(path))`. Return the path. If not a directory, log warning and return path anyway (container may not have source mounted yet at startup — validation is best-effort).
     - `GITHUB_URL`: call `cloneOrPull()` and return the clone directory path.
   - `private String cloneOrPull()`:
     - `Path cloneDir = Path.of(config.getCloneDirectory());`
     - If `cloneDir.resolve(".git")` exists:
       - Open existing repo: `Git.open(cloneDir.toFile())`
       - Verify remote URL matches `config.getGithubUrl()` via `git.getRepository().getConfig().getString("remote", "origin", "url")`. If mismatch, delete directory and re-clone.
       - `git.pull().setCredentialsProvider(credentials()).call()`
     - Else: `Files.createDirectories(cloneDir)`, then `Git.cloneRepository().setURI(config.getGithubUrl()).setDirectory(cloneDir.toFile()).setBranch("refs/heads/" + config.getBranch()).setCredentialsProvider(credentials()).call()`
     - Return `cloneDir.toString()`
   - `private UsernamePasswordCredentialsProvider credentials()`: return `new UsernamePasswordCredentialsProvider("x-token", config.getGithubToken())`
   - `public boolean isResolved()`: returns `resolvedSourceRoot != null && !resolvedSourceRoot.isBlank()`
