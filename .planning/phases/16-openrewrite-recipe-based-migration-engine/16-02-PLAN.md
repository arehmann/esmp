    **4. Create MigrationRecipeService** (`com.esmp.migration.application`):
    - `@Service` annotation
    - Constructor injects: `Neo4jClient neo4jClient`, `JavaSourceParser javaSourceParser`, `SourceAccessService sourceAccessService`
    - Private helper: `loadActionsFromGraph(String classFqn)`:
      ```cypher
      MATCH (c:JavaClass {fullyQualifiedName: $fqn})-[:HAS_MIGRATION_ACTION]->(ma:MigrationAction)
      RETURN ma.actionType AS actionType, ma.source AS source, ma.target AS target,
             ma.automatable AS automatable, ma.context AS context
      ```
      Returns `List<MigrationActionEntry>`
    - Private helper: `loadActionsForModule(String module)`:
      ```cypher
      MATCH (c:JavaClass)-[:HAS_MIGRATION_ACTION]->(ma:MigrationAction)
      WHERE split(c.packageName, '.')[2] = $module
      RETURN c.fullyQualifiedName AS classFqn, c.sourceFilePath AS sourceFilePath,
             collect({actionType: ma.actionType, source: ma.source, target: ma.target,
                      automatable: ma.automatable, context: ma.context}) AS actions
      ```
    - `public MigrationPlan generatePlan(String classFqn)`:
      - Load actions from graph
      - Split into automatable (YES) and manual (PARTIAL + NO) lists
      - Compute automationScore: (yesCount + 0.5 * partialCount) / total
      - Return MigrationPlan record
    - `public MigrationResult preview(String classFqn)`:
      - Call generatePlan to get automatable actions
      - If no automatable actions, return MigrationResult.noChanges()
      - Load source file path from Neo4j: `MATCH (c:JavaClass {fullyQualifiedName: $fqn}) RETURN c.sourceFilePath`
      - Parse source file into LST using `javaSourceParser.parse(List.of(path), projectRoot, null)`
      - Build composite recipe from automatable actions via `buildCompositeRecipe()`
      - Execute: `InMemoryExecutionContext ctx = new InMemoryExecutionContext(t -> log.warn("Recipe error", t));`
      - `RecipeRun run = recipe.run(new LargeSourceSet(lsts), ctx);` -- NOTE: check actual API. If `LargeSourceSet` is not available, use `recipe.run(lsts, ctx)` where lsts is `List<SourceFile>`. The research doc's API may not match 8.74.3 exactly. Verify with actual imports.
      - Collect Result: `run.getChanges()` or `run.getResults()` depending on API version
      - Return MigrationResult with diff() and modified source printAll()
    - `public MigrationResult applyAndWrite(String classFqn)`:
      - Call preview() to get result
      - If hasChanges, write modifiedSource to sourceFilePath via `Files.writeString()`
      - Return result
    - `public BatchMigrationResult applyModule(String module)`:
      - Load all classes with actions in the module from Neo4j
      - Filter to classes with automatable actions (automationScore > 0)
      - For each class, call applyAndWrite()
      - Collect results, count modifications, track errors
      - Return BatchMigrationResult
    - Private helper: `buildCompositeRecipe(List<MigrationActionEntry> actions)`:
      - Create `List<Recipe> recipes = new ArrayList<>()`
      - For each action:
        - `CHANGE_TYPE` -> `new ChangeType(action.source(), action.target(), false)` (false = not just rename, full type change)
        - `CHANGE_PACKAGE` -> `new ChangePackage(action.source(), action.target(), false)`
        - Other types -> log warning and skip (future extension point)
      - If `recipes.isEmpty()` return null
      - Wrap in a named recipe: create an anonymous `Recipe` subclass that returns the list from `getRecipeList()`. Or use `org.openrewrite.Recipe` composition pattern.
      - NOTE on OpenRewrite 8.74.3 API: `ChangeType` is in `org.openrewrite.java`. `ChangePackage` is in `org.openrewrite.java`. Both accept (oldFqn, newFqn, boolean ignoreDefinition). Use `new ChangeType(old, new, true)` where ignoreDefinition=true to avoid modifying the target type's own definition.

    **5. Create MigrationValidationQueryRegistry** (`com.esmp.migration.validation`):
    - Extends `ValidationQueryRegistry`
    - `@Component` annotation
    - 3 validation queries:
      - `MIGRATION_ACTIONS_POPULATED`: `MATCH (ma:MigrationAction) RETURN count(ma) AS count` — passes if count > 0
      - `MIGRATION_SCORES_COMPUTED`: `MATCH (c:JavaClass) WHERE c.migrationActionCount > 0 AND c.automationScore IS NOT NULL RETURN count(c) AS count` — passes if count > 0
      - `MIGRATION_ACTION_EDGES_INTACT`: `MATCH (c:JavaClass)-[:HAS_MIGRATION_ACTION]->(ma:MigrationAction) WITH c, count(ma) AS edgeCount WHERE edgeCount <> c.migrationActionCount RETURN count(c) AS count` — passes if count = 0 (all edge counts match)
    - Total validation queries becomes 44 (41 + 3)
