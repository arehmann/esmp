**1. Create SchedulingService** at `src/main/java/com/esmp/scheduling/application/SchedulingService.java`:

**Main method: `ScheduleResponse recommend(String sourceRoot)`**

Step 1 — Aggregate module metrics via Neo4jClient Cypher:
```java
String cypher = """
    MATCH (c:JavaClass)
    WHERE c.packageName IS NOT NULL AND size(c.packageName) > 0
    WITH split(c.packageName, '.')[2] AS module, c
    WHERE module IS NOT NULL
    WITH module,
         count(c) AS classCount,
         avg(coalesce(c.enhancedRiskScore, 0.0)) AS avgEnhancedRisk,
         avg(coalesce(c.complexitySum, 0.0)) AS avgComplexity,
         max(coalesce(c.complexityMax, 0)) AS maxComplexity
    RETURN module, classCount, avgEnhancedRisk, avgComplexity, maxComplexity
    """;
```
Map results to a `Map<String, ModuleMetrics>` (private inner record: `module, classCount, avgEnhancedRisk, avgComplexity, maxComplexity`).

Step 2 — Get dependent counts per module via Cypher:
```java
String dependentCypher = """
    MATCH (c1:JavaClass)-[:DEPENDS_ON]->(c2:JavaClass)
    WHERE c1.packageName IS NOT NULL AND c2.packageName IS NOT NULL
    WITH split(c1.packageName, '.')[2] AS sourceModule,
         split(c2.packageName, '.')[2] AS targetModule
    WHERE sourceModule IS NOT NULL AND targetModule IS NOT NULL AND sourceModule <> targetModule
    WITH targetModule AS module, count(DISTINCT sourceModule) AS dependentCount
    RETURN module, dependentCount
    """;
```
Map to `Map<String, Integer>` (module -> dependentCount). Modules not in this map have 0 dependents.

Step 3 — Get cross-module dependency edges (for topological sort):
```java
String edgeCypher = """
    MATCH (c1:JavaClass)-[:DEPENDS_ON]->(c2:JavaClass)
    WHERE c1.packageName IS NOT NULL AND c2.packageName IS NOT NULL
    WITH split(c1.packageName, '.')[2] AS sourceModule,
         split(c2.packageName, '.')[2] AS targetModule
    WHERE sourceModule IS NOT NULL AND targetModule IS NOT NULL AND sourceModule <> targetModule
    RETURN DISTINCT sourceModule, targetModule
    """;
```
Build `Map<String, Set<String>> dependencies` where key = sourceModule, value = set of targetModules it depends on.

Step 4 — Git frequency:
```java
Map<String, Integer> commitCounts = gitFrequencyService.computeModuleCommitCounts(
    sourceRoot, weightConfig.getGitWindowDays());
```

Step 5 — Topological sort (Kahn's algorithm):
- Initialize inDegree map and dependents (reverse adjacency) map for all modules from Step 1
- For each edge (source depends on target): increment inDegree of source, add source to dependents list of target
- BFS by wave: seed queue with modules where inDegree == 0, process wave by wave, decrement inDegrees
- After BFS: any module NOT in `moduleWave` map is in a cycle — assign to `maxWave + 1`

Step 6 — Compute composite scores:
- Find maxDependentCount, maxCommitCount, maxComplexity across all modules (guard: if max == 0, set to 1 to avoid division by zero)
- For each module, compute:
  - `normalizedRisk = avgEnhancedRisk` (already [0,1])
  - `normalizedDependency = Math.min(1.0, (double) dependentCount / maxDependentCount)`
  - `normalizedFrequency = Math.min(1.0, (double) commitCount / maxCommitCount)`
  - `normalizedComplexity = Math.min(1.0, Math.log(1 + avgComplexity) / Math.log(1 + maxComplexity))`
  - Guard: if `Math.log(1 + maxComplexity) == 0`, set normalizedComplexity to 0
- `riskContribution = weightConfig.getRisk() * normalizedRisk`
- `dependencyContribution = weightConfig.getDependency() * normalizedDependency`
- `frequencyContribution = weightConfig.getFrequency() * normalizedFrequency`
- `complexityContribution = weightConfig.getComplexity() * normalizedComplexity`
- `finalScore = riskContribution + dependencyContribution + frequencyContribution + complexityContribution`

Step 7 — Build rationale string per module:
Format: `"Avg risk %.2f (%s), %d dependents, %d commits/%dmo, avg CC %.1f — %s"`
Where risk label: < 0.3 "safe", < 0.6 "moderate", else "high"
Where summary: finalScore < 0.3 "safe early target", < 0.6 "moderate priority", else "complex — schedule later"

Step 8 — Assemble ModuleSchedule records, group into WaveGroup by waveNumber, sort within each wave by finalScore ascending.
- `flatRanking`: all ModuleSchedule sorted by waveNumber ASC then finalScore ASC
- `waves`: List<WaveGroup> sorted by waveNumber ASC
- `generatedAt`: `Instant.now().toString()`
- `durationMs`: measured from method start
